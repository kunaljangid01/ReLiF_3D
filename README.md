# ReLiF-3D: Prior-Guided Semi-Supervised 3D MRI Segmentation via Robust Bias-Consistent Paired Views

[![MICCAI 2026](https://img.shields.io/badge/MICCAI-2026-blue)](#)
[![Project Page](https://img.shields.io/badge/Project-Page-green)](https://visdomlab.github.io/ReLiF_3D/)
[![License](https://img.shields.io/badge/License-Research%20Only-lightgrey)](#license)

Official implementation of **ReLiF-3D**, accepted at **MICCAI 2026**.

> **ReLiF-3D** is a prior-guided semi-supervised framework for robust 3D MRI segmentation under extreme label scarcity. It uses a **frozen SAM-Med3D** foundation model as a stable anatomical prior and trains a lightweight **3D U-Net** with bias-consistent paired views generated by **Smooth Orthogonal Bias Field (SOBF)** augmentation. Reliability-aware constraints, **CGCR** and **LARC**, stabilize unlabeled optimization and improve lesion-focused representation learning.

**Project page:** https://visdomlab.github.io/ReLiF_3D/
**Paper:** https://visdomlab.github.io/ReLiF_3D/paper.html
**Code:** https://github.com/kunaljangid01/ReLiF_3

---

## Highlights

- **Prior-guided semi-supervised learning:** uses frozen SAM-Med3D as a stable training-time prior instead of an EMA teacher.
- **SOBF paired-view generator:** simulates scanner-like smooth MRI bias using orthogonal multiplicative and additive fields.
- **CGCR:** confidence-gated co-regularization to reduce unreliable consistency updates.
- **LARC:** lesion-aware representation consistency guided by the frozen foundation prior.
- **Low-label 3D segmentation:** evaluated on BraTS 2019 and Left Atrium (LA) with 1, 2, 3, and 5 labeled volumes.

---

## Framework Overview

ReLiF-3D contains two branches during training:

1. **Frozen foundation branch:** SAM-Med3D provides stable anatomical/lesion priors.
2. **Trainable segmentation branch:** a lightweight 3D U-Net learns task-specific segmentation using labeled and unlabeled volumes.

During inference, **only the 3D U-Net is used**. SAM-Med3D is not required at deployment.

## Training

ReLiF-3D is trained under extremely low-label settings with:

```text
S ∈ {1, 2, 3, 5} labeled volumes
remaining training volumes used as unlabeled data
input crop size: 128 × 128 × 128
batch size: 2
maximum iterations: 30000
optimizer: SGD
learning rate: 0.01
weight decay: 1e-4
momentum: 0.9
```

### Example: BraTS 2019

```bash
cd code
python ReLiF_3D_train_brats.py \
  --prompt mask \
  --root_path ../data/BraTS19 \
  --output_dir ReLiF_3D \
  --labeled_num 2 \
  --batch_size 2 \
  --exp BraTS/ReLiF_3D

```

### Example: Left Atrium

```bash
cd code
python ReLiF_3D_train_la.py \
  --prompt mask \
  --root_path ../data/2018LA_Seg_Training_Set \
  --output_dir ReLiF_3D \
  --labeled_num 2 \
  --exp LA/ReLiF_3D \
  --batch_size 2


```

## Reproducing Paper Results

For each dataset, run the training script with different values of `--labeled_num`:

```bash
--labeled_num 1
--labeled_num 2
--labeled_num 3
--labeled_num 5
```

Recommended practice:

1. Use the same labeled/unlabeled splits as the paper.
2. Fix random seeds for reproducibility.
3. Report results as mean ± standard deviation.
4. Use the provided evaluation script for Dice, Jaccard, 95HD, and ASD.
